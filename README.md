 ### Описание проекта:

:honeybee: Вторая часть дипломного проекта

**Разработан и выполнен SQL-запрос для проверки целостности данных курьеров.**
Проведена валидация отображения созданных заказов в базе данных путем составления запроса, который выводит список логинов курьеров с подсчетом количества заказов, находящихся в статусе «В доставке».

**Реализована выгрузка статусов заказов по заданной бизнес-логике.**
Создан скрипт для выборки всех трекеров заказов с соответствующими статусами. Преобразование статусов выполнено согласно правилам:

*finished == true → статус 2;*

*cancelled == true → статус -1;*

*inDelivery == true → статус 1;*

во всех остальных случаях → статус 0.

**Разработан и реализован автоматизированный API-тест для проверки жизненного цикла заказа.**
Написан автотест, покрывающий следующий сценарий:

✿ Отправлен запрос на создание нового заказа.

✿ Из ответа сохранен трек-номер заказа.

✿ Выполнен запрос к системе для получения деталей заказа по данному треку.

В ходе проверки подтверждено, что код ответа при получении данных равен 200, что свидетельствует об успешной обработке запроса и наличии данных в системе.
